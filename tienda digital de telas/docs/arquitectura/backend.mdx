## Descripción

El backend de **D&D Textil** es una API REST nativa construida en Java plano, usando el servidor HTTP integrado del JDK (`HttpServer`) y GSON para serialización. No utiliza frameworks como Spring Boot, optimizando su peso y demostrando un dominio puro del lenguaje.

## Endpoints

El `ApiServer` levanta escuchas bajo puertos estándar (ej: `8080`) a los siguientes Contextos REST:

- `/api/login` - Verifica Email y Hash SHA-256 (Manejado por `LoginHandler`).
- `/api/products` - Orquesta el Catálogo CRUD (GET, POST, PUT, DELETE) (Manejado por `ProductsHandler`).

### Gestión CORS

El servidor inyecta automáticamente cabeceras CORS preflight (`OPTIONS`) en todas las respuestas para permitir comunicación desde los orígenes de desarrollo de Vite (ej: localhost:3000 o localhost:5173).

### Dependencias

El bloque Backend Java requiere estrictamente:
- `gson-2.10.1.jar`
- `postgresql-42.7.3.jar`

Ambos deben mapearse al compilar o al invocar `java -cp`.

## Patrón Estructural

1. `ApiServer.java`: Entry point / Orquestador.
2. `Handlers`: Interpretan Request, gestionan Body JSON y preparan Respuestas (HTTP 200, 401, 500).
3. `DAO`: Objetos de Acceso a Datos (Ej. `ProductDAO.java`, `UserDAO.java`), que ejecutan puramente SQL transaccional.
