## ProductGrid & ProductCard

El `Catalog.jsx` es la espina transversal por donde ocurre la conversión. Consume a los objetos del Endpoint y los mapea en `ProductCard`.

- **Animación:** Los componentes `Card` usan variables físicas para deslizarse en hover sin romper la estética *Glassmorphism*.
- **Sync de Imágenes:** Anteriormente Unsplash (mock), las previsualizaciones ahora leen `product.images[0]` directo de PostgreSQL/Storage en tiempo real, incluso durante un Checkout o en la Wishlist del cliente.

## CartContext 

Las peticiones de adición a cesta de compras (`/carrito`) no llaman al backend. Se procesan de forma transiente, se agrupan por ID, subtotal, SKU, y se congelan en Local Storage. Solamente en la página de `/checkout` se vacían mediante procesamientos.

```javascript
const handleAddToCart = (e) => {
    e.preventDefault();
    addToCart(product, quantity); // Función provista de CartContext
};
```
