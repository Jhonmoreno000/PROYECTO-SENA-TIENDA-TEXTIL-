---
title: "Autenticación (AuthContext)"
description: "Gestión segura del estado de usuario (JWT/Storage)"
---

La Autenticación maneja la sesión en memoria y delega su persistencia al `localStorage`.

### Login (`AuthContext.login`)

Al enviar las credenciales, un `fetch` a `/api/login` recibe las cabeceras REST. Si el backend responde HTTP 200, los datos del usuario logueado en crudo se guardan en el contexto para inyectarlos en Componentes y Guardias.

```jsx
// Flujo estándar dentro de cualquier página
const { user, login, logout, isLoading } = useAuth();
```

### Hash Criptográfico

La seguridad en base de datos guarda TODAS las contraseñas en Hash `SHA-256`. Las validaciones del login crudo transforman los bytes a hexadecimal ANTES de cruzar información en memoria en Java. 

> **Nota Crítica:** Los usuarios con cuentas previas (Demo) deben correr el parche de SHA-256 de las tablas o de lo contrario no podrán iniciar sesión, por el desajuste alfanumérico.
